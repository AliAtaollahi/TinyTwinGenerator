# TinyTwinGenerator

TinyTwinGenerator generates a tiny twin `.aut` model from a Rebeca-generated state-space file and a list of observable actions.

The script produces two output files:

```text
tinytwin.aut
tinytwin.dot
```

## Requirements

This project uses:

- Lingua Franca compiler (`lfc`)
- mCRL2 tools, especially `ltsconvert`
- Python 3
- GCC/G++
- CMake
- Make

The versions used during development were:

```text
lfc 0.10.0

ltsconvert mCRL2 toolset 202407.1.7448c7f6f9 (Release)

Python 3.8.10

g++ 11.4.0
gcc 11.4.0

cmake 4.1.2

GNU Make 4.2.1
```

## Build

To build the Lingua Franca components, run:

```bash
./cleanup_build.sh --build samples/heater.statespace observable_actions.txt
```

This compiles:

```text
castfunction_variables
extraction_function
```

After you build once, you do not need to use `--build` again unless you change the `.lf` files.

## Run Without Rebuilding

After the first build, run:

```bash
./cleanup_build.sh samples/heater.statespace observable_actions.txt
```

## Specify Output Directory

By default, the output files are written to the current directory.

To specify an output directory, use `-o` or `--output`:

```bash
./cleanup_build.sh -o output samples/heater.statespace observable_actions.txt
```

or:

```bash
./cleanup_build.sh --output output samples/heater.statespace observable_actions.txt
```

The generated files will be:

```text
output/tinytwin.aut
output/tinytwin.dot
```

## Build and Specify Output Directory

You can also build and specify the output directory at the same time:

```bash
./cleanup_build.sh --build -o output samples/heater.statespace observable_actions.txt
```

## Input Files

The script expects two input files:

```text
<input.statespace> <observable_actions.txt>
```

Example:

```bash
./cleanup_build.sh samples/heater.statespace observable_actions.txt
```

`observable_actions.txt` should contain the actions that must remain observable in the tiny twin.

## Output Files

The final files are:

```text
tinytwin.aut
tinytwin.dot
```

`tinytwin.aut` is the generated tiny twin in AUT format.

`tinytwin.dot` is the DOT representation of the generated tiny twin.

## Intermediate Files

Intermediate files are stored in:

```text
build/
```

This directory is automatically regenerated by the script.
